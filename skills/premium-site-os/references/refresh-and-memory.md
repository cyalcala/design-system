# Refresh and Memory

## Freshness Protocol

- Re-scan trend sources for major projects.
- Treat trend notes older than 90 days as stale.
- Treat repo rankings older than 30 to 60 days as stale.
- Treat implementation docs older than 180 days as needing review.
- Record date checked beside trend/source claims.

## Taste Memory Entry

Record:

- Pattern name
- Status: Worked | Failed | Stale | Needs retest
- Audience
- Premium mode
- Why it worked or failed
- Evidence
- When to use
- When to avoid
- Refresh date

Promote repeated wins into reusable patterns. Demote overused patterns into anti-slop warnings.
