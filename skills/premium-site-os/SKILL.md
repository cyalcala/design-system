# Premium Site OS

## Overview

Use this skill to turn messy design inputs into a premium website direction, build packet, or QA pass. Keep the user-facing flow simple; use the deeper references only when needed.

## Core Workflow

1. **Intake**: identify business, audience, offer, goal, sources, permissions, and desired feeling.
2. **Source read**: classify inputs as lead, support, constraint, rejection, trend signal, or do-not-use.
3. **Taste direction**: choose one premium mode and anti-reference list.
4. **Art bible**: define typography, palette, layout, components, motion, mobile, accessibility, and anti-slop rules.
5. **Storyboard**: plan the page from opening promise to final conversion moment.
6. **Motion plan**: choose one signature motion family and restrained support motion.
7. **Build packet or QA**: produce the requested artifact and note source safety.

## Non-Negotiables

- Do not copy third-party code, copy, logos, images, videos, or distinctive brand expression without rights.
- Treat third-party sites and clone repos as pattern sources, not deployable client templates.
- Keep all real interactions as semantic HTML/CSS/JS, not baked into images or videos.
- Plan mobile and reduced-motion behavior before implementation.
- Use one signature moment. Do not turn every section into a demo reel.
- If trend or repo data may be stale, browse and record the date checked.

## Output Modes

- **Quick Concept**: source read, premium mode, art direction, section storyboard, signature moment.
- **Premium Concept**: source scorecard, trend scan, art bible, variants, build packet, QA rubric.
- **Production Build**: implementation plan/site, verification, performance/accessibility/mobile checks, deployment notes.
