# Awwwards Commercial Safety Rules

Date: 2026-05-21

## Purpose

Awwwards-style references are valuable, but many are clones, tutorials, or educational rebuilds. This doc keeps Premium Site OS commercially safe.

## Golden Rule

Learn patterns. Do not copy expression.

## Allowed

- Study public websites and repos.
- Extract high-level layout principles.
- Learn animation architecture.
- Study scroll choreography.
- Learn implementation patterns from appropriately licensed code.
- Rebuild a user's own site closely when authorized.
- Create original designs inspired by multiple references.

## Not Allowed Without Rights

- Copy third-party code into client work.
- Reuse images, videos, logos, music, fonts, or illustrations.
- Copy exact copywriting.
- Preserve another brand's trade dress.
- Recreate a distinctive clone so closely that it substitutes for the original.
- Claim an educational clone as original client work.

## License Checks

Before adapting repo code:

1. Check license.
2. Check README disclaimers.
3. Check asset ownership.
4. Check whether the project says educational only.
5. Prefer rewriting the pattern from scratch.

If there is no license:

- Treat code as not reusable.
- Use the repo for study only.

## Distance Rules

### Near

Allowed for:

- User-owned sites
- Client-authorized rebuilds
- internal prototypes

### Medium

Allowed for:

- category conventions
- common component patterns
- non-distinctive interaction ideas

### Far

Default for:

- competitor references
- Awwwards winners
- clone repos
- third-party brand sites

## Adaptation Checklist

Before client delivery:

- Brand identity is different.
- Copy is original.
- Assets are owned, licensed, or generated for the project.
- Layout is meaningfully transformed.
- Signature moment is original or sufficiently reinterpreted.
- Code is original or license-compatible.
- No reference site could be confused with the final site.

## How To Use Clone Repos Safely

Use them to answer:

- How is the animation structured?
- How are sections sequenced?
- How is the timeline coordinated?
- How does mobile simplify?
- What performance risks appear?
- What pattern can be rebuilt in our own style?

Do not use them to answer:

- What exact visuals should we ship?
- What assets should we reuse?
- What exact page should we deploy?
