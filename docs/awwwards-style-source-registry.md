# Awwwards-Style Source Registry

Date checked: 2026-05-21

## Purpose

This registry tracks source repos and references that can teach Awwwards-style motion, layout, and production patterns.

Use this as a learning registry, not a cloning list.

## Safety Position

Many Awwwards-style repos are clones, rebuilds, or tutorials inspired by real award-winning sites. They can teach motion architecture and interaction patterns, but client work must transform the ideas into original expression.

Default rule for third-party sources:

- Learn the pattern.
- Rebuild the principle.
- Do not copy identity, assets, copy, code, or distinctive creative expression.

## Ranking Rubric

Do not rank by stars alone. Score 1 to 5:

- Visual polish
- Animation sophistication
- Production readiness
- Code learnability
- Mobile quality
- Performance posture
- Originality safety
- License/commercial safety
- Pattern transfer value

## What To Extract

- Animation architecture
- Scroll choreography
- Asset sequencing
- Section pacing
- Text reveal timing
- Responsive motion fallbacks
- Component structure
- Performance tradeoffs

## What Not To Extract

- Exact brand identity
- Source assets
- Copywriting
- Logos
- Exact layouts
- Distinctive clone-specific moments
- Unlicensed code

## Registry Maintenance

- Re-check repo metadata every 30 to 60 days.
- Demote inactive or low-quality repos.
- Promote sources only after a working case study proves transfer value.
- Record date checked beside every score.
