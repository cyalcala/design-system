# Disciplines, Methods, and Powers Register

Date: 2026-05-17

## Purpose

This register lists the current disciplines, methods, and powers inside Premium Site OS.

Open Design is treated as an optional prototype/design lab. The system remains useful without it.

## Executive Summary

Yes, the disciplines and methods are coherent. The system now has enough structure to avoid being just a collection of AI tricks.

The strongest operating model is:

> Sources to strategy to art direction to design bible to build packet to QA to taste memory.

The danger is not lack of power. The danger is exposing too much complexity too early. The working interface should remain simple:

> Send links, images, assets, and a goal. Get a premium direction or build packet.

## Core Disciplines

### 1. Business Diagnosis

Before visuals, define the commercial truth:

- Audience
- Offer
- Primary conversion
- Emotional target
- Trust signals
- What should feel premium
- What the site must never feel like

Purpose: prevent "pretty but pointless."

### 2. Source Intelligence

Classify each input before using it:

- Lead source
- Support source
- Constraint source
- Rejection source
- Trend signal
- Do-not-use source

Purpose: bad images, weak links, or irrelevant references should not control the direction.

### 3. Reference Transformation

Use references to learn, not copy.

Allowed:

- Study structure
- Extract pacing
- Learn component patterns
- Understand conversion logic
- Rebuild owned or authorized sites closely

Not allowed:

- Copy third-party source code
- Copy logos, copy, images, video, or brand identity
- Recreate distinctive expression too closely

Purpose: preserve originality and reduce legal/brand risk.

### 4. Trend and Taste Intelligence

Scan current design/taste signals before major projects.

Trend signals are weighted, not obeyed.

Purpose: prevent stale output while avoiding trend-chasing.

### 5. Premium Mode Selection

Choose one dominant design lane:

- Cinematic luxury
- Editorial founder brand
- High-density SaaS
- Immersive scroll journey
- Boutique studio
- Product launch spectacle
- Technical command center
- Wellness / ritual / calm commerce

Purpose: make the site feel intentional.

### 6. Art Bible Discipline

Convert taste into rules:

- Typography
- Palette
- Spacing
- Section density
- Component behavior
- Image treatment
- Motion grammar
- Mobile rules
- Accessibility rules
- Anti-slop rules

Purpose: keep the whole site coherent.

### 7. Storyboard Discipline

Plan the page like a film, not a random list of sections:

1. Opening promise
2. World/problem
3. Mechanism
4. Product/service demonstration
5. Proof
6. Offer/pricing
7. Objection handling
8. Final conversion moment

Purpose: make the site persuasive and paced.

### 8. Motion Restraint

Motion must have a job:

- Reveal meaning
- Guide attention
- Create emotional pace
- Show transformation
- Make the brand memorable
- Support scroll narrative

Purpose: prevent animation from becoming decoration.

### 9. Anti-Slop Critique

Detect and remove generic AI patterns:

- Generic centered SaaS hero
- Purple-blue gradient blobs
- Fake glassmorphism everywhere
- Nested card piles
- Flat typography
- Vague AI copy
- Random reveal animations
- Fake UI baked into images

Purpose: protect premium perception.

### 10. Proof and Learning Discipline

Every serious project should create evidence:

- Baseline AI output
- Premium Site OS output
- Before/after screenshots
- QA scores
- Lessons learned
- Taste memory entries

Purpose: prove the system works and improve over time.

## Methods

### Tiny First Workflow

1. Intake
2. Source read
3. Creative direction
4. Build packet
5. Output

Purpose: usable immediately.

### One-Image Method

Turn one image into:

- Meaning layer
- Composition layer
- System layer
- Interaction layer
- Responsive hero
- Art bible
- Signature moment

Purpose: extract a full brand world from one strong visual.

### URL Deconstruction Method

For links:

1. Capture desktop/mobile.
2. Map section order.
3. Identify conversion path.
4. Extract component inventory.
5. Note typography, palette, motion, density, and spacing.
6. Separate functional pattern from distinctive expression.
7. Decide reference distance.

Purpose: learn from sites without blindly cloning them.

### Authorized Clone / Rebuild Method

For owned/client-authorized websites:

1. Inventory content and assets.
2. Preserve what matters.
3. Improve design, performance, accessibility, and conversion.
4. Modernize the system.

Purpose: upgrade existing sites safely.

### Visual Composition Seed Method

Prompt for a composition system, not a website template.

Generate:

- Full desktop reference
- Clean desktop background
- Mobile reference
- Clean mobile background
- Motion prompt
- Negative-space map

Purpose: create original visual direction.

### Background Plate Method

Remove baked-in UI from generated visuals.

Purpose: keep buttons, nav, forms, CTAs, and interactions as real HTML.

### Mobile-First Asset Method

Create mobile references and crop rules early.

Purpose: prevent beautiful desktop / broken mobile.

### Image-to-Video Motion Method

Animate static visuals into subtle loops:

- Terminal flicker
- Steam
- Light movement
- Particles
- Interface activity
- Scroll-linked video

Purpose: create cinematic premium energy.

### Variant Lab Method

Generate variants only for high-leverage sections:

- Hero
- Pricing/offer
- Proof/testimonials
- Feature explanation
- Final CTA
- Signature interaction

Purpose: improve taste through comparison without creating chaos.

### Signature Moment Method

Add one memorable interaction:

- Living terminal hero
- Scroll-controlled video journey
- 3D card procession
- Product reveal
- Portal/vault transition
- Kinetic typography
- Interactive before/after

Purpose: create the premium multiplier.

### Build Packet Method

Compile all decisions into one handoff:

- Business brief
- Source inventory
- Trend scan
- Premium mode
- Art bible
- Assets
- Section storyboard
- Responsive rules
- Motion plan
- Anti-slop checklist
- QA rubric

Purpose: give Codex or another builder enough context to execute well.

### Before/After Proof Loop

Compare normal AI output against Premium Site OS output using the same rubric.

Purpose: validate that the system beats generic AI site generation.

### Taste Memory Method

Save:

- What worked
- What failed
- What became stale
- What clients liked
- What users responded to
- What implementation traps repeated

Purpose: make the system compound.

## Powers

### 1. Image-to-Website Power

One image can seed a full website direction.

### 2. Link-to-Direction Power

A website URL can be deconstructed for structure, interaction, conversion logic, and visual principles.

### 3. Authorized Clone Power

Owned or authorized sites can be rebuilt closely and improved.

### 4. Reference Transformation Power

Third-party references can inspire original designs without copying identity, code, assets, or exact expression.

### 5. Transcript-to-Method Power

Design walkthroughs can become reusable workflows, prompts, and QA gates.

### 6. Trend-Aware Taste Power

The system can refresh taste decisions with current signals.

### 7. Art Bible Power

Visual taste becomes durable rules.

### 8. Variant Comparison Power

Multiple directions can be compared before committing.

### 9. Anti-Slop Power

Generic AI design patterns can be detected and removed.

### 10. Motion-as-Product Power

Animation becomes storytelling, not decoration.

### 11. Responsive Asset Power

Desktop, tablet, and mobile are planned together.

### 12. Build Packet Power

Messy inspiration becomes a clear execution packet.

### 13. Proof Loop Power

The system can prove whether it beats baseline AI output.

### 14. Taste Memory Power

The system can learn from repeated projects and avoid stale patterns.

### 15. Cost/Time Mode Power

The same system can operate in:

- Quick Concept
- Premium Concept
- Production-Ready Build

### 16. Client Intake Power

Non-designers can provide useful inputs through a simple form.

### 17. Hermes Memory Power

Optional future layer: Hermes can help remember patterns, maintain taste memory, and turn repeated workflows into skills.

### 18. Open Design Lab Power

Optional tool layer: Open Design can serve as a design/prototype lab with design systems, skills, responsive preview, and export.

## Tool Roles

### Premium Site OS

The judgment layer: strategy, taste, reference rules, build packet, QA.

### Codex

The production builder: code, implementation, verification, repo workflow.

### Open Design

Optional prototype lab: visual exploration, design systems, responsive previews, exportable artifacts.

### Impeccable-Style Critique

Anti-slop critic: typography, layout, color, motion, UX writing, polish.

### Hermes

Optional memory layer: taste memory, repeated skills, scheduled research, learning loops.

### Notion

Knowledge backup: methodology, decisions, summaries, case studies.

### GitHub

Versioned source of truth for docs, templates, and eventually code.

## Current Assessment

The disciplines are sound. They cover strategy, inputs, taste, references, implementation handoff, QA, proof, and memory.

The main risk is complexity. The mitigation is to keep two layers:

- **Simple user interface**: "Send links/images/goal. Get premium direction."
- **Deep internal system**: scoring, trend scan, art bible, variants, QA, taste memory.
